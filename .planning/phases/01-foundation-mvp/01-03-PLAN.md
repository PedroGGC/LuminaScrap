# Plan 01-03: Spec Parser Pipeline + Price Normalization

**Phase:** 1 (Foundation & MVP)
**Status:** Pending execution

## Objective

Build the spec parser to extract CPU, GPU, RAM, storage from product titles. Implement Brazilian price parsing and create test suite.

## Requirements Covered

- PARSE-01: Extract CPU, GPU, RAM, storage (type + size), marca from title
- PARSE-02: Regex + lookup for ~80% coverage, Claude API fallback
- PARSE-03: Test suite for parser with real Brazilian product titles
- PARSE-04: Price normalization ("R$ 1.234,56" → integer centavos)

## Tasks

### 3.1 Regex Parser Core
```
[TASK] Build regex-based spec parser
  - CPU detection: i3/i5/i7/i9, Ryzen 3/5/7/9, generations (12th, 13th, etc.)
  - GPU detection: RTX 4050/4060/4070/4080/4090, GTX series, integrated
  - RAM detection: 8GB/16GB/32GB/64GB, DDR4/DDR5
  - Storage detection: 256GB/512GB/1TB/2TB, SSD/HDD/NVMe
  - Brand detection: Dell, Samsung, Avell, Lenovo, Asus, etc.
```

### 3.2 Lookup Tables
```
[TASK] Create CPU/GPU lookup mappings
  - CPU family to generation mapping
  - GPU tier mapping (RTX 4090 > 4080 > 4070...)
  - Brand normalization (lenovo = Lenovo)
```

### 3.3 Brazilian Price Parser
```
[TASK] Implement price normalization function
  - Input: "R$ 1.234,56" or "R$1234,56"
  - Output: 123456 (integer in centavos)
  - Handle edge cases: ".", ",", spaces
```

### 3.4 Claude API Fallback
```
[TASK] Implement Claude API fallback for complex titles
  - Batch titles that regex can't parse
  - Call Claude API with structured prompt
  - Store parsed results
  - Handle API errors gracefully
```

### 3.5 Parser Test Suite
```
[TASK] Create parser tests
  - 20+ real product titles from Kabum/ML
  - Cover edge cases: gaming, office, workstation
  - Measure coverage (target: 80%+)
```

## Threat Model

- **API cost:** Claude API calls cost money — limit to edge cases, cache results
- **Data quality:** Bad parser = bad data — invest in test coverage

## Dependencies

- `zod` for schema validation
- `openai` or `@anthropic-ai/sdk` for Claude fallback

## Success Criteria

- [ ] Parser extracts specs from 80%+ of product titles
- [ ] Price normalization handles all Brazilian formats
- [ ] Test suite validates parser accuracy
- [ ] Claude fallback handles edge cases
