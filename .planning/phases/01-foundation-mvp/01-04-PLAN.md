# Plan 01-04: Fastify API Server + Render Deploy

**Phase:** 1 (Foundation & MVP)
**Status:** Pending execution

## Objective

Build the Fastify REST API with endpoints for notebooks, prices, and manual scrape trigger. Deploy to Render.

## Requirements Covered

- API-01: GET /notebooks with filters (CPU, GPU, RAM, marca, price range)
- API-02: GET /notebooks/:id for detail
- API-03: GET /prices/:id for price history
- API-04: POST /scrape for manual trigger
- API-05: Scalar docs at /docs

## Tasks

### 4.1 Fastify Server Setup
```
[TASK] Initialize Fastify server
  - Install: fastify, @fastify/cors, @fastify/type-provider-typebox
  - Create app.ts with basic config
  - Add health check endpoint
```

### 4.2 GET /notebooks Endpoint
```
[TASK] Implement notebooks listing with filters
  - Query params: cpu, gpu, ramMin, ramMax, brand, priceMin, priceMax, page, limit
  - Use Drizzle queries with filters
  - Implement sorting (price, name, updated)
  - Return paginated results
```

### 4.3 GET /notebooks/:id Endpoint
```
[TASK] Implement notebook detail
  - Fetch single notebook by ID
  - Include latest price from prices table
  - Return full spec details
```

### 4.4 GET /prices/:id Endpoint
```
[TASK] Implement price history
  - Fetch all prices for product
  - Sort by scraped_at desc
  - Return array of {valor, scraped_at}
```

### 4.5 POST /scrape Endpoint
```
[TASK] Implement manual scrape trigger
  - Accept: { sources: ["kabum", "ml"] | "all" }
  - Trigger scraper functions
  - Return job ID for polling (future: webhook)
```

### 4.6 Scalar API Documentation
```
[TASK] Add interactive API docs
  - Install @scalar/fastify-api-reference
  - Configure at /docs route
  - Document all endpoints with examples
```

### 4.7 Render Deploy
```
[TASK] Deploy API to Render
  - Create Render service (web)
  - Set environment variables (Neon connection)
  - Configure build: npm run build
  - Set start: npm run start
```

## Threat Model

- **API security:** No auth in MVP — consider rate limiting
- **Data exposure:** Public API — ensure no sensitive data in responses
